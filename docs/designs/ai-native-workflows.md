## Vision

### 10x Check
An AI workflow platform that learns what each agent needs, suggests automations proactively, executes with full context awareness, and gets smarter over time. The workflow page becomes the agent's automation command center — not a list of saved macros, but a living system that understands their business. Natural language in, automated actions out. Every tool the AI has is automatically available in every workflow.

### Platonic Ideal
A new agent opens CloAgent. They go to Workflows. Instead of a form with dropdowns, they see a clean prompt: "What would you like to automate?" They describe their ideal workflow in plain English. The AI refines it through conversation, breaks it down into steps, shows a visual preview, and saves it. When triggered, the AI agent executes the plan using its 34 tools — adapting to each contact's context. If the output isn't right, the agent says "make it more casual" and the AI updates the workflow. The feeling: delegating to a competent assistant who remembers your preferences.

## Accepted Scope (added to this plan)

### Core
- Natural language workflow creation via AI chat conversation
- AI-powered refinement (AI asks clarifying questions, presents plan, gets confirmation)
- AI-native execution: each workflow run spawns a focused AI agent session with the workflow instruction + context, using all 34 existing tools
- Workflow CRUD (overhaul existing form-based system)

### Expansions (from original plan)
- Scheduled triggers (cron-like: "every Monday at 8am", "30 days after close")
- Conversational editing ("make the email more casual" via AI chat)
- Dry run mode (preview what a workflow would do without executing)
- Batch execution (run a workflow for multiple contacts/deals matching criteria)
- Run notifications + activity feed integration + approval modes (review-before-execute for write actions)
- Visual step timeline (beautiful step visualization replacing the current text list)

### Expansions (from CEO review — SCOPE EXPANSION mode)
- **Save as Workflow** from any AI conversation — one-click to convert chat actions into a reusable workflow (Effort S)
- **Conversational error recovery** — when a workflow step fails, spawn a mini-conversation for the agent to decide how to proceed (Effort M)

- **AI becomes workflow-aware** — add workflow list to AI system prompt so it can suggest and reference existing workflows (Effort S)
- **Workflow analytics** — success rates, execution times, "saved you X hours" dashboard panel (Effort M)

## Key Architecture Decision

**Workflows = saved AI instructions.** The `instruction` field (natural language prompt) IS the workflow. The `steps` field is a visualization aid generated by the AI during creation, not the execution plan. At runtime, the AI agent reads the instruction and reasons about what to do — it's not replaying a fixed sequence.

This means:
- Every new AI tool is automatically available in workflows (zero workflow-specific code)
- Workflows can adapt to context (different contact → different email content)
- The system gets better as the AI model improves
- Cost: ~$0.01-0.05 per run (Haiku 4.5) — acceptable for the flexibility gained

## Key Design Decisions (from CEO review)

1. **Kill old workflow engine** — Remove `workflow_engine.py`. Zero users = zero migration risk. Recreate 4 templates as AI-native workflows. One execution path, not two.
2. **Approval mode is agent-controlled** — The real estate agent sets `approval_mode` per workflow (review/auto). Auto mode still logs all actions to `workflow_runs.step_results` and activity feed for audit trail.
3. **Approach B selected** — Full AI-Native plan, not minimal fix (A) or platform (C).
