## Vision

### 10x Check
The 10x version is a daily intelligence briefing dashboard — agents open CloAgent to a prioritized action queue ranked by lead score and urgency. Each action is a card with contact name, score, reason, and a one-click action button. The AI has already reasoned about the entire pipeline and organized the agent's day.

### Platonic Ideal
The agent opens CloAgent and feels like they have a brilliant assistant who reviewed every contact overnight. The home screen shows prioritized recommendations with context. The score isn't a number — it's the engine behind an action queue that tells you exactly what to do. Each recommendation has a "Do it" button that opens AI chat pre-loaded with context.

## Scoring Model

### 4-Dimension Heuristic (0-100)

| Dimension | Max | What it measures |
|-----------|-----|-----------------|
| Engagement | 30 | Weighted activity count with time decay (14d full, 15-30d half, >30d zero) |
| Readiness | 30 | Deal stage progression + pre-approval + budget + timeline |
| Velocity | 20 | Activity acceleration (last 7d vs prior 7d) |
| Profile Completeness | 20 | Buyer profile fields filled (budget, locations, property type, etc.) |

### Score Tiers

| Tier | Range | Color | Meaning |
|------|-------|-------|---------|
| Hot | 80-100 | Green | Ready to close, needs immediate attention |
| Warm | 50-79 | Yellow | Active and progressing, keep nurturing |
| Cool | 20-49 | Blue | Some interest, needs re-engagement |
| Cold | 0-19 | Gray | Inactive, consider archiving or drip campaign |

### Activity Type Multipliers
- Showing: 1.5x
- Call: 1.0x
- Email: 0.75x
- Note/Task: 0.5x

### Implementation

- **Computation:** Inline `computeLeadScore(tx pgx.Tx, contactID string)` runs within existing RLS-scoped transactions
- **Storage:** `lead_score INTEGER`, `lead_score_signals JSONB`, `previous_lead_score INTEGER` on contacts table
- **Fault tolerance:** Score computation errors are logged but writes succeed — activity/deal/profile mutations are never blocked by scoring failures
- **AI integration:** New `get_lead_score` read tool (#35: 19 read + 16 write) returns score, tier, dimension breakdown, top signals, and suggested action

### Trigger Points
Score recomputes on: activity create/delete, deal stage change/delete, buyer profile create/update, contact create/update

### Eng Review Additions (2026-03-25)
- **Going cold banner trigger:** `lead_score < 20 AND last activity > 14 days ago` — shows "N contacts going cold" count on contacts list
- **Kill switch:** `show_lead_scores` boolean in user settings (default true) — frontend checks before rendering any score UI
- **Implementation notes:** DeleteDeal must fetch contact_id BEFORE delete for score recompute. CreateGeneralActivity skips recompute when contact_id is nil.

### Design Review Additions (2026-03-25)

#### Score Drill-Down (Contact List)
- **Interaction:** Clicking a score badge on the contact list opens a **slide-over panel from the right (400px)**
- **Panel contents:** Dimension breakdown (Engagement/Readiness/Velocity/Profile sub-scores), AI explanation, suggested action button
- **Dismissal:** Click outside panel or press Escape
- **On contact detail page:** Inline expandable card below the header badge (no slide-over)

#### Score Change Indicators (↑↓)
- **Placement:** 12px arrow positioned immediately right of the 42px score badge, outside the circle
- **Colors:** Green ↑ for increase of 5+ points, red ↓ for decrease of 5+ points
- **Threshold:** Hidden if absolute change < 5 points (prevents noise from trivial fluctuations)
- **Data source:** Compares `lead_score` vs `previous_lead_score` column

#### Pipeline Kanban Badge
- **Size:** Compact 24px circle, 10px font (vs 42px on contact list)
- **Placement:** Next to existing health dot on deal cards
- **Content:** Score number + tier color only. No signal tags on Kanban cards.

#### Interaction States
- **Score badge (new contact, score = 0):** Show "0" in Cold tier gray styling. Score IS zero — that's accurate.
- **Score computation:** No loading state needed — computation is synchronous within DB transaction.
- **AI explanation in slide-over (loading):** Show dimension breakdown immediately (from `lead_score_signals` JSONB). Skeleton pulse in AI text area while generating.
- **AI explanation (error/unavailable):** Show breakdown + signal tags. Replace AI text with "AI analysis temporarily unavailable."
- **Going cold banner (0 cold contacts):** Banner hidden entirely.
- **Going cold banner (1 contact):** "1 contact is going cold" (singular).
- **Going cold banner (N contacts):** "N contacts are going cold" (plural).
- **Going cold banner dismiss:** X button, dismissed for 24 hours (localStorage TTL, matching follow-up banner pattern).
- **Kill switch off (`show_lead_scores = false`):** Hide score column from contact list, remove "Sort by score" option, hide score badge from contact detail header, hide going cold banner, hide score badges on Kanban cards. AI `get_lead_score` tool remains available.

#### Signal Tags
- **Selection:** Top 3 signals by contribution weight from `lead_score_signals` JSONB
- **Overflow:** "+N" chip if more than 3 signals
- **Styling:** Green (`#f0fdf4`/`#16a34a`/`#bbf7d0`) = positive, Amber (`#fffbeb`/`#d97706`/`#fde68a`) = warning, Gray (`#f1f5f9`/`#64748b`/`#e2e8f0`) = neutral

#### Sort Behavior
- **Default sort unchanged** — keep existing contacts page default sort order
- **New option:** Add "Score" to sort dropdown. When active, sorts descending by `lead_score`

#### Suggested Action Button
- **Style:** Primary filled button (not text link) in the slide-over panel
- **Behavior:** Clicking opens contact-scoped AI chat with the suggested action as a pre-filled message

#### Brief Me Integration
- **Approach:** Prompt engineering only — inject score data into Brief Me system prompt
- **No new UI:** AI naturally weaves score context into briefing response text
