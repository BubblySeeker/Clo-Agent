**2. Modify the agent loop tool dispatch (around line 371):**
Replace the current `if tool_name in WRITE_TOOLS:` / `elif tool_name in READ_TOOLS:` block with a THREE-branch check. The auto-execute check MUST come FIRST (before the general WRITE_TOOLS check) because auto-execute tools are a subset of WRITE_TOOLS:

```python
            if tool_name in WRITE_TOOLS and tool_name in AUTO_EXECUTE_TOOLS:
                # Auto-execute safe write tools without confirmation
                logger.info("Auto-executing %s for agent %s", tool_name, agent_id)
                result = await _dispatch_write_tool(tool_name, tool_input, agent_id)
                status = "error" if "error" in result else "success"
                yield sse({"type": "auto_executed", "name": tool_name, "result": result, "status": status})
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": tb.id,
                    "content": json.dumps(result, default=str),
                })
            elif tool_name in WRITE_TOOLS:
                # Dangerous write tools — queue for user confirmation
                confirmation = queue_write_tool(tool_name, tool_input, agent_id)
                yield sse({"type": "confirmation", **confirmation})
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": tb.id,
                    "content": json.dumps({"status": "pending_confirmation", "pending_id": confirmation["pending_id"]}),
                })
            elif tool_name in READ_TOOLS:
                result = await execute_read_tool(tool_name, tool_input, agent_id)
                yield sse({"type": "tool_result", "name": tool_name, "result": result})
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": tb.id,
                    "content": json.dumps(result, default=str),
                })
            else:
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": tb.id,
                    "content": json.dumps({"error": f"Unknown tool: {tool_name}"}),
                })
```

The rest of the loop (the `has_confirmation` check after the for-loop, lines 396-411) stays EXACTLY the same. This is critical: when a mixed round has both auto-execute and confirmation tools, the auto-execute tools run normally within the for-loop, but when the confirmation tool is hit it also adds a pending_confirmation result. After the for-loop, `has_confirmation` detects it, streams the note, and breaks. This matches the spec: "Auto-execute tools run normally. When a confirm tool is hit, queue it and break as usual."
